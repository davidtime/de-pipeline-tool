Operator Validation

In the Operator Library, you have the option of creating your own validation rules. These rules determine whether a property's input is valid or not. Validation types can be broken up into three categories:

  • Text Input - This type allows for user input through a text box and validates off of regular expressions. It also allows for multiple inputs on one property if needed
  • File Input - This type has users upload a file. It matches based off the file extension if you list one as a requirement
  • Select Box - This type gives the user options they have to choose from for a property. In Validation Rules, you can either define your own options or you can upload a mapping file to incorporate them for you

We also provide five basic rules to use for your own properties:

  • String - A text input field that accepts any non empty input
    • Regular Expression: /.+/
  • Integer - A text input field that accepts only integers
    • Regular Expression: /^\-?\d+/
  • FloatingPoint - A text input field that accepts any number
    • Regular Expression: /^\-?\d+(\.\d+)?/
  • StringList - A text input field that supports multiple entries and accepts any nonempty input
    • Regular Expression: /.+/
  • File - A file input field that allows for any type of file to be uploaded

Validation Rules

Under validation rules, you can create your own property rules like the ones above. Like the previous lists, you can add and remove rules using the provided controls.

  • Name - The name of the rule
  • Type - The type of the rule
  • RegEx - The regular expression used to evaluate the property
  • List - Whether or not multiple inputs are allowed

  • Extension - The file extension accepted by the rule

  • Import Mapping - This allows you to choose a mapping file from Schema Wizard and upload it. The contents are then used for select box entries
  • Options -The current options the user has for that property
