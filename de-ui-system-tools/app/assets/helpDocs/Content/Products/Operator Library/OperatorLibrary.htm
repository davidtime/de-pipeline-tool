Operator Library

The operator library stores and lets you control all your operators for use in the System Builder.

  • New operators can be added using the plus button in the upper right hand corner, and they can be deleted with the trashcan button on each row. Clicking a row selects that operator

Details

Similar to the System Manager, the details pane provides the operator details. Unlike the system manager however, you can customize just about anything regarding that operator here.

  • name - The operator name
  • className - The name and path of the java file within the far
  • displayname - The name the operator displays in the System Builder
  • type - The type of operator
  • jar - This lets you choose which jar to use with this operator. You can use an existing jar if there was already one uploaded or you can upload a new one
  • jarName - The name of the jar uploaded
  • jar_file - This is the option to upload a new jar (it is not visible is "Use Existing Jar" is selected)

Properties

The second half of the details pane lists all the properties the operator has. Like the operator list, you can add and remove properties using the same kind of controls

  • Property Name - The name of the property
  • Display Name - The name displayed for the property in the System Builder
  • Required - Whether or not the property is required by the operator
  • Validation Rule - What set of guidelines are used to determine if a property is valid or not

Note that if you do change an operator and save it, old operators in a system will not be affected and must be recreated. This is to prevent issues where old properties that are filled out are matched with new validation rules. However, if you have a stored operator, it will be updated with the new validation.
