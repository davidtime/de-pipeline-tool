System Manager

The system manager allows you to view all current systems built in the builder. From here, you can control the system and view diagnostic information about it.

System List

The system list shows you every system that has been built and saved on the System Builder. There are two different views: tile and detail.

Tile view better visually displays what state a system is, but it doesn't provide as much information on its own.

The buttons on the left control the system.

  • Power - Turns a system on
  • Cross - Turns a system off
  • Blocks - Opens the system in the System Builder
  • Trash - Deletes the system

 

The list view offers the same buttons and statuses, but the statuses aren't as apparent. Instead, it offers start and end times. It is also sortable by any of the columns.

System Monitor

The monitor gives you a general look at how your system is running. There are two graphs it provides: a line graph that displays tuples processed and emitted, and a pie graph that shows processor usage.

  • PSMA - Per second minute average (the average number of tuples over a minute average

You can also toggle the monitoring using the Pause/Start Monitoring button in the upper right hand corner. By default, monitoring is turned off.

Details

The details tab shows the properties of the running system.

  • name - The fullsystem name you assigned in the builder
  • state - The current state of the system
  • startedTime - The last time the system was started
  • finishedTime - The last time the system finished or turned off

Logging

This tab shows any errors that the system has run into. It lists the operator where the error occurred and the error received. There are controls to clear the current logs for that system and to temporarily stop the logging in case there is a large influx of large you need to get through. You can also pop out the individual error into a new window to help for debugging purposes.
